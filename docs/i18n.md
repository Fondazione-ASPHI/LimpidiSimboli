# js/i18n.js — Traduzioni e internazionalizzazione

**Righe:** ~998 | **Dipendenze:** `utils.js` (usa `els`)

## Responsabilità

- Contenere **tutte** le stringhe visibili all'utente nelle tre lingue (IT, EN, ES).
- Fornire funzioni per applicare traduzioni al DOM e gestire messaggi di stato.

## Strutture dati

### `statusMessages`

```js
const statusMessages = {
  translate_ita_en_failed: {
    it: 'Attenzione: la traduzione ITA→ENG per "{term}" non è riuscita...',
    en: '...',
    es: '...'
  },
  // ~40 chiavi totali
};
```

Oggetto con ~40 chiavi di stato. Ogni valore è un oggetto `{it, en, es}` con stringhe template che supportano placeholder `{param}`.

**Chiavi principali**: `translating`, `search_error`, `no_results`, `opensymbols_token_missing`, `js_error`, `unhandled_rejection`, `generating_image`, `analyzing_sentence`, `explain_loading`, ecc.

### `translations`

```js
const translations = {
  it: { appTitle: 'Limpidi Simboli', translateButton: '🔄 Converti in simboli', ... },
  en: { appTitle: 'Clear Symbols', translateButton: '🔄 Convert to symbols', ... },
  es: { appTitle: 'Símbolos Claros', translateButton: '🔄 Convertir en símbolos', ... }
};
```

Oggetto principale con ~100+ chiavi per lingua. Copre:
- Titoli e sottotitoli dell'app
- Etichette di tutti i pulsanti
- Placeholder dei campi input
- Testo dei modal (impostazioni, guide, esercizi)
- Stringhe del footer (con HTML)
- Legenda delle funzioni simbolo
- Messaggi alert/prompt/confirm

**Proprietà aggiunte singolarmente** (da ~riga 500 in poi): chiavi aggiuntive vengono assegnate direttamente a `translations.it`, `translations.en`, `translations.es` per stringhe aggiunte in fasi successive dello sviluppo.

## Funzioni

### `formatStatusText(template, params)` → `string`
Sostituisce placeholder `{chiave}` nel template con i valori corrispondenti dall'oggetto `params`.

```js
formatStatusText('Cercando "{term}"...', {term: 'gatto'})
// → 'Cercando "gatto"...'
```

### `renderLegend()`
Costruisce dinamicamente la legenda delle funzioni nel `<details id="symbolsLegendWrapper">`. Legge le chiavi `legend_*` da `translations`.

### `setStatusKey(key, params, isError)`
Imposta il messaggio di stato localizzato nella barra `#live`:
1. Cerca `key` in `statusMessages`.
2. Seleziona la lingua corrente da `els.lang.value`.
3. Applica `formatStatusText` con `params`.
4. Imposta `els.live.textContent` e la classe CSS `error` se `isError`.

### `setStatus(text, isError)`
Wrapper retrocompatibile: imposta direttamente `els.live.textContent`. Usare `setStatusKey` per nuovi messaggi.

### `applyTranslations(langObj)`
Applica tutte le traduzioni al DOM:
- Elementi con `data-i18n="chiave"` → `textContent = langObj[chiave]`.
- Elementi con `data-i18n-html="chiave"` → `innerHTML = langObj[chiave]`.
- Elementi con `data-i18n-placeholder="chiave"` → `placeholder = langObj[chiave]`.
- Aggiorna anche titoli ARIA e attributi speciali.

### `translateUI(lang, params)` → `string`
Ritorna la stringa tradotta per la lingua corrente. Supporta placeholder.

```js
translateUI('translateButton')  // → '🔄 Converti in simboli' (se lang=it)
```

## Come aggiungere una nuova stringa

1. Aggiungere la chiave in `translations.it`, `translations.en`, `translations.es`.
2. Nel HTML, usare `data-i18n="nuovaChiave"` sull'elemento.
3. Se è un messaggio di stato, aggiungerlo a `statusMessages` e usare `setStatusKey('nuovaChiave', {})`.
4. Se contiene HTML, usare `data-i18n-html="nuovaChiave"` nel DOM.

## Come aggiungere un nuovo messaggio di stato

1. Aggiungere una nuova chiave a `statusMessages`:
   ```js
   nuova_chiave: {
     it: 'Messaggio con {param}...',
     en: 'Message with {param}...',
     es: 'Mensaje con {param}...'
   }
   ```
2. Chiamare: `setStatusKey('nuova_chiave', { param: valore })`.
